# Agent Index Design System

## Design Read

Agent Index 是一个高密度的产品情报工具。设计应让读者在办公室环境中长时间阅读、检索和核验资料，视觉感受安静、可信、敏捷。

## Dials

- Design variance: 5
- Motion intensity: 3
- Visual density: 6

## Color

- 使用层叠石墨深色工作区作为默认主题，通过边界、透明表面和低对比空间流场建立深度。
- 暖琥珀色是唯一强调色，只用于当前状态、主要行动和真实数据提示。
- 不使用装饰渐变、彩色能力评分或没有数据依据的视觉编码。

## Typography

- 使用系统无衬线字体与系统等宽字体，优先保证中文与英文混排稳定性，避免远程字体加载闪烁。
- 正文基础字号 16px，紧凑元数据不低于 12px，移动端正文不低于 14px。
- 标题字距不低于 -0.04em，正文行宽控制在 75ch 内。

## Layout

- 最大内容宽度 1280px。
- 首页使用不对称双栏介绍核心价值和真实能力覆盖数据。
- 首页核心视觉为基于真实竞争格局的动态产品星图，使用克制的 WebGL 雷达扫描强化“持续监测”语义；国外按本地桌面与云端企业路线区分，国内突出腾讯、阿里、字节和模型厂商。
- 产品索引与时间线使用固定筛选区加可扫描内容区。
- 详情页使用主内容加辅助事实栏，并在窄屏退化为单栏。

## Components

- 卡片圆角 10px，控件圆角 8px，标签使用小型圆角，不使用大胶囊卡片。
- 卡片以边界、间距和背景分组，阴影仅用于悬浮反馈且模糊不超过 8px。
- 按钮高度至少 44px，具备 hover、active 与 focus-visible 状态。
- 状态点只用于“每周自动更新”这一真实状态。
- 品牌标志使用内联 SVG 的索引行图形，保持高分屏清晰度，不依赖运行时替换。

## Motion

- 仅使用轻微的悬浮位移与颜色过渡。
- 允许与真实数据结构绑定的持续流场、雷达扫描、节点呼吸与信号传播；不使用与内容无关的背景视频。
- 不使用整页滚动渐入、数字计数或弹性动画；动效只表达交互状态。
- 尊重 `prefers-reduced-motion`。
