# 重大动态收录规则

本网站不是完整更新日志，只收录会改变产品竞争判断的事件。

## 正式收录条件

候选事件至少满足一项：

- 发布新产品、重大版本或新的核心工作模式
- 新增 Computer Use、多 Agent、长期记忆、IM/跨端入口、企业治理等关键能力
- 定价、开放范围、商业模式发生明显变化
- 重要合作、停止运营或品牌整合，并且直接改变 Agent 产品入口、能力或交付方式
- 多家可信媒体集中报道，形成明显行业声量

以下内容通常不收录：Bug 修复、普通体验优化、小范围 UI 调整、常规模型升级、没有实际产品变化的营销内容、收购融资、股价财报、估值传闻、纯公司战略或资本动作。

收购、融资、财务、股价、估值、IPO、公司战略调整等新闻，即使与目标公司或产品名称相关，也默认不进入正式页。只有当新闻同时包含明确的 Agent 产品发布、核心能力上线、可用入口变化或停止运营信息时，才作为产品动态重新评估。

## 更新颗粒度

自动任务宁可多进候选池，也不直接多上正式页。正式页只呈现筛过后的内容：

- 现有产品：优先收录会改变用户使用方式、产品定位或竞争格局的更新，例如新的执行入口、记忆/反思能力、多 Agent 协作、云端/本地执行模式、企业治理与连接器生态。
- 新产品：必须是独立产品或独立入口，不把单个模型、普通 API、一次性营销活动当成产品入库。
- 模型更新：只有当它直接改变某个 Agent 产品能力或商业化入口时才收录；单纯模型榜单、参数、跑分不进入正式页。
- 媒体聚合源：AIBase、AI工具集、AI HOT 只作为发现渠道；正式收录仍尽量追溯到官方公告或可信一手链接。

## 信源优先级

1. 官方公告、官方博客、官方产品页、GitHub Release
2. 厂商官方社交账号和公开演讲
3. 可信行业媒体或主流科技媒体
4. 社区讨论仅用于发现线索，不能单独作为正式来源

中文聚合源的定位：

- AIBase、AI工具集、AI HOT 用于补足中文 AI 产品新闻发现，不默认代表最终事实来源。
- 微信公众号文章可作为发现源或官方社交账号信源，但自动采集依赖微信读书登录态，默认不在 GitHub Actions 中无人值守运行。

正式事件必须保留来源链接和核验日期。

## 无人值守自动审核

每周任务采用保守发布策略，只有以下候选会自动进入正式时间线：

- 官方一手信源，候选得分不低于 `6`
- 可信主流或行业媒体，候选得分不低于 `9`，且标题明确包含目标产品名称
- 可信中文聚合源或媒体，标题明确包含目标产品名称，并且属于长期记忆、多 Agent、Computer Use、企业治理等核心能力变化，候选得分不低于 `6`

标题与产品不匹配，或属于评论、传闻、市场行情、财务资本动作、日常更新的候选会自动拒绝。相关但证据不足的候选会自动暂缓，不会污染正式网站。每次运行都会在 `reviews/` 保存审核记录。

GitHub Release 只根据发布标题判断事件强度，不使用整篇 Release Notes 评分，避免普通版本因说明中罗列大量能力而被误判为重大更新。

## 候选评分

- 核心产品发布、停止运营：`+6`
- Computer Use、多 Agent、长期记忆、跨端入口、企业治理：`+4`
- 定价、合作、开放范围变化：`+3`
- 标题明确包含发布、上线、推出、宣布等事件词：`+2`
- 仅包含修复、优化、改进等日常词：`-4`
- 收购、融资、财务、股价、估值、IPO、纯战略动作：自动审核阶段直接拒绝

自动任务默认只保留得分不低于 `5` 的候选，再使用更严格的无人值守发布阈值审核。

## 新产品入库

新产品候选必须确认它是独立产品，而不是已有产品的一项普通功能；同时需要找到官方一手信源，并补齐厂商、官网、产品类型和核心定位后才能加入产品库。当前无人值守任务会自动暂缓新产品候选，避免仅凭媒体标题错误入库。
