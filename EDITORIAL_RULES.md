# 重大动态收录规则

本网站不是完整更新日志，只收录会改变产品竞争判断的事件。

## 正式收录条件

候选事件至少满足一项：

- 发布新产品、重大版本或新的核心工作模式
- 新增 Computer Use、多 Agent、长期记忆、IM/跨端入口、企业治理等关键能力
- 定价、开放范围、商业模式发生明显变化
- 重要合作、融资、收购、停止运营或品牌整合
- 多家可信媒体集中报道，形成明显行业声量

以下内容通常不收录：Bug 修复、普通体验优化、小范围 UI 调整、常规模型升级、没有实际产品变化的营销内容。

## 信源优先级

1. 官方公告、官方博客、官方产品页、GitHub Release
2. 厂商官方社交账号和公开演讲
3. 可信行业媒体或主流科技媒体
4. 社区讨论仅用于发现线索，不能单独作为正式来源

正式事件必须保留来源链接和核验日期。自动采集只生成候选，不直接发布。

## 候选评分

- 核心产品发布、收购、停止运营：`+6`
- Computer Use、多 Agent、长期记忆、跨端入口、企业治理：`+4`
- 定价、融资、合作、开放范围变化：`+3`
- 标题明确包含发布、上线、推出、宣布等事件词：`+2`
- 仅包含修复、优化、改进等日常词：`-4`

自动任务默认只保留得分不低于 `5` 的候选，最终是否收录由人工判断。

## 新产品入库

新产品候选必须确认它是独立产品，而不是已有产品的一项普通功能；同时需要找到官方一手信源，并补齐厂商、官网、产品类型和核心定位后才能加入产品库。
